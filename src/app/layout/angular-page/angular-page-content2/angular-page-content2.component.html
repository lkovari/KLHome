Act #1 Custom Input Component with Validation
Formerly I faced the following problem, how can I include custom components into the NgForm validation mechanism. I wanted to see the custom component as a FormControl in the collection of controls of form, and would like to see it is participating in the Form validation mechanism. I spent a lot of time to investigate how can I do that, I found in the Angular documentation the ControlValueAccessor interface.

Interested in this technology very much me, therefore I spent really a lot of time in my spare time (usually at night) for exploring and practicing how can I implement it, in the best way. I would like to implement correctly the ControlValueAccessor.

At first I created a simple input component the class name of it TextInputComponent whic is the plain usage of the ControlValueAccessor the selector of the component is 'app-text-input'.
Act #2 Custom Date Component I wrapped a third party Calendar component which created by PrimeNG.
The class name of this component is CustomCalendarComponent the selector of this component is 'app-custom-calendar'. I took a look to the PrimeNG source code, and I noticed the original third party component is also implemented the ControlValueAccessor. Of course I know to wrap a component without good reason, is a dangerous venture, because the developer should consider what parameters move to the wrapper component and what not, because we can hide the wrapped component parameters. I should also mention, although this is not an exhaustive implementation, but the aspect of implementation of the ControlValueAccessor it will be more than enough.
Act #3 A simple Form with the Components TextInputComponent and CustomCalendarComponent which I mentioned above.
I just created a simple form with two input fields, one of is the text input component and the other is the date calendar component. The Submit button will enable if the form is valid which means, the field content is mathes with validation rules. Usage of the setValues(form) and clearValues(form) are the best example how can we load the data into the fields before the form open and how can be clear about that.
The validation rules are the following the
  • Custom Date field is required
  • Custom Text field is required, minimum length is {{minLength}} characters and the first character should be capital letter
